## 13-round review cycle — the headline learning

This release used a dual-Opus paired-lens review cycle that ran 13 times before convergence. Each round had two parallel reviewers with different lenses. Real blockers found per round:

| Round | Blockers found | Class |
|---|---|---|
| 1 | 1 | Plan fidelity (integration + stale literals) |
| 2 | 2 | Prod-hardening (reindexFn wiring, raw-file lock) |
| 3 | 2 | DoS (unbounded transcript) + orphan summary |
| 4 | 2 | CVE-class exploits (header injection + timestamp poison) |
| 5 | 1 | Straggler — um-tunnel had same sed regression as session-start.sh |
| 6 | 0 | Contract parity (11 schema_version sites + doc drift) |
| 7 | 2 | Docs lied about flock coordination + 2 flags silently ignored |
| 8 | 2 | **Unit tests never ran in CI** + hollow flag receivers |
| 9 | 2 | Docker-broken config path + reindexFn signature mismatch |
| 10 | 1 | **Summary files had no frontmatter — reindex silently failed; core value broken** |
| 11 | 1 | Token-cost baseline drift + smoke cleanup-loop parse |
| 12 | 0 | **First convergence round** |
| 13 | 1 | `<TBD>` placeholders in user-facing docs |

**Why so many rounds?** Each round applied a distinct lens (plan-fidelity, prod-hardening, end-to-end, adversarial, test-quality, docs-code alignment, supply-chain, cross-system integration, convergence-validation, external-reviewer simulation). Blockers found through one lens were invisible from another. The pattern that kept surfacing: **tests passed with DI-injected mocks while the production wiring was silently broken** (R2 reindex wiring hollow, R9 signature mismatch, R10 frontmatter missing). Only cross-system integration review (R10) caught the frontmatter case; only live-stack smoke (Z.2) validated R10's fix.

**Was 13 rounds too many?** For a complex release touching MCP tools, REST endpoints, installer, CI, Docker, and cross-language concurrency — no. Rounds 1-11 each found 1-3 real blockers. Rounds 12-13 confirmed convergence with minimal findings. The paired-lens discipline (two Opus reviewers per round with different angles) doubled the chance of catching each blocker.

**Cost:** ~1.3M tokens of Opus review across 26 paired dispatches + ~500k for fix-application subagents. Dollars: ~$200 in Opus. For a core-value-path release, cheap.

## Key lessons

1. **Test discipline alone is not enough.** R3/R4/R5-class bugs (hollow wiring, signature mismatch, silent frontmatter) had passing tests that asserted the wrong contract. Spy-based DI tests can encode the broken behavior they're trying to prevent. The fix: pair unit tests with live smoke that exercises the REAL dispatcher + REAL mem0 + REAL Docker.

2. **Review-lens diversity > review count.** Applying 5 rounds of the same lens would miss what R10's cross-system integration lens caught (summary frontmatter).

3. **Document reality, not plans.** R7's CHANGELOG/MIGRATION correction (flock coordination claim) was a direct consequence of the code admitting the gap in-comment while docs hid it.

4. **CI is documentation.** R8's discovery that unit tests never ran in CI despite being committed was a ~2-month blind spot. Every claim "N tests pass" needs a CI artifact backing it.

5. **Paired-Opus review has a clear ROI curve.** Each round found 1-3 blockers through round 11; rounds 12-13 confirmed convergence. The knee is clear — stop when both reviewers say SHIP without new findings.
