# Benchmarks

Every number universal-memory publishes is generated by its own eval harness, measures a layer UM built (or the end-to-end product, labeled as such), and is regenerable by the command shown. We deliberately publish **no** numbers for capabilities owned by the engines we build on — vector-search accuracy, retrieval ranking, and index storage belong to [mem0](https://github.com/mem0ai/mem0) and [Qdrant](https://github.com/qdrant/qdrant); session-compression numbers belong to [claude-mem](https://github.com/thedotmack/claude-mem). See their repositories for their numbers, and the README's ["Built on proven foundations"](../README.md#built-on-proven-foundations) section for how the pieces relate.

Figures are point-in-time stamped. Regenerating: run the command, commit the updated results JSON, update this file — drift shows up in diffs.

---

## Recall latency — end-to-end (labeled)

**p50 211 ms / p95 354 ms on a production Raspberry Pi 5** (72-fact live corpus); **p50 210 ms / p95 344 ms on x86** (Ryzen 7 9800X3D, 300-fact seeded corpus). N=100 timed `memory_search` calls per leg, full HTTP round-trip of the MCP tool path, 5 discarded warmups.

> Measured **end-to-end against UM's API** — the figure includes the per-search OpenAI embedding call, network to OpenAI, the vector engine, and UM's own handler. It is a product-experience number, not an engine benchmark; latency is dominated by the embedding round-trip, which is why the Pi and a desktop CPU tie.

- Command: `node --env-file=.env eval/recall-latency-eval.mjs --mode seed-and-measure --corpus 300 --out eval/results/<date>-recall-latency-dev.json` (dev) / `--mode measure-only --pace-ms 1100` against a live instance (Pi)
- Evidence: [`server/eval/results/2026-07-10-recall-latency-b4-dev.json`](../server/eval/results/2026-07-10-recall-latency-b4-dev.json), [`…-b4-pi.json`](../server/eval/results/2026-07-10-recall-latency-b4-pi.json) — run 2026-07-10, embedding model `text-embedding-3-small`

## Fact-extraction fidelity

**Precision 1.00, recall 0.98–1.00, noise abstention 7/8** on UM's 40-row extraction benchmark (gpt-4.1-nano extractor, gpt-4o-mini judge). The sub-1.00 recall reading is a judge-granularity artifact, audited on every occurrence to date: the extractor emits a compound fact carrying two gold propositions and the judge matches it to one — **zero information loss** in all audited runs. The one tolerated noise miss (a hedged future read as committed) is documented in the v1.5.2 release notes.

> Scope: UM's own extraction prompt and pipeline on UM's fixture — not a general-purpose extraction benchmark.

- Command: `node --env-file=.env eval/extraction-fidelity-eval.mjs --fixture eval/extraction-set.jsonl --out eval/results/<date>-extraction.json`
- Evidence: [`server/eval/results/2026-07-10-extraction-b1.json`](../server/eval/results/2026-07-10-extraction-b1.json) (run 2026-07-10) + the v1.5.2 five-run baseline

## Duplicate-write handling

**100% of identical re-writes merged (20/20), 0 false merges on distinct controls (10/10), 0 store growth on a full-corpus rewrite** — the idempotency invariant a memory store lives or dies by. Reported with full honesty: light **paraphrases** of the same fact merged only 27.8% of the time at the default embedding threshold (0.84) — exact-duplicate handling is airtight; paraphrase-level dedup is an open improvement area, tracked as future work rather than hidden.

- Command: `node --env-file=.env eval/dedup-effectiveness-eval.mjs --out eval/results/<date>-dedup.json` (live server required)
- Evidence: [`server/eval/results/2026-07-10-dedup-effectiveness-b3.json`](../server/eval/results/2026-07-10-dedup-effectiveness-b3.json), 50-row fixture, run 2026-07-10

## Session-checkpoint cost

**$0.000155 median per checkpoint** (p95 $0.000167; median 391 tokens in / 160 out) — the LLM cost of synthesizing a session's state-of-play, measured over 20 independent fresh-session checkpoints (each walled into its own project so no transcript accumulation inflates or deflates the number). Model mix: gpt-4.1-nano extraction + the configured summarizer.

- Command: `node --env-file=.env eval/checkpoint-cost-eval.mjs --n 20 --out eval/results/<date>-checkpoint-cost.json` (live server required)
- Evidence: [`server/eval/results/2026-07-10-checkpoint-cost-b2.json`](../server/eval/results/2026-07-10-checkpoint-cost-b2.json), run 2026-07-10
